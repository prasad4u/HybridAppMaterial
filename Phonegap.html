WHAT IS PHONEGAP

Allows you to build applications for mobile devices using JavaScript, HTML5, and CSS3, instead of device-specific languages such as Objective-C or Java.

HYBRID APPLICATIONS

  • PhoneGap creates what is now called hybrid applications
  • They are not web applications, as they can use device functionalities such as Camera, File Systems etc. They are also packaged/bundled and can be made available on application stores.
  • They are not native applications per se as they use HTML and CSS to render their layouts as opposed to using native UI frameworks, e.g. Cocoa.
  • Application logic is written with Javascript as opposed to a primary language of choice for the target platform, that would be Objective-C for iOS or Java for Android.

INSTALLATION : DEPENDENCIES

  • First, Install Node.js (http://nodejs.org/download/)
  • It is available for Mac, Linux and Windows
  • Go to you command line and run following command to test if you have successfully installed it
    npm --version

INSTALLING : PHONEGAP CLI

  • Run following command in your command line
    sudo npm install -g phonegap
  • This will install 'phonegap' command that we we use predominantly to create phonegap based apps
  • Once again, use following command to verify that installation has been successful
    phonegap --version

INSTALLATION : IOS SPECIFIC

  • On your Mac, Install XCode & Command Line Tools for XCode. Linux and Windows users can ignore this step
  • Install 'ios-sim' command. This command is used to launch iOS simulator from command line.
    npm install -g ios-sim

INSTALLATION : ANDROID SPECIFIC

  • Download latest Android Studio
  • Applicable for Mac, Linux and Windows.
  • Install it at your preferred location on your machine
  • Update 'PATH' to include ' /sdk/tools/'. This will allow 'android' command to be available to your shell
  • Update 'PATH' to include ' /sdk/platform-tools/'. This will allow 'adb' command to be available to your shell

.cordova folder

  • A typical convention followed to store configuration related to a command. In this case the command is cordova
  • The .cordova folder holds the configuration at project level. Hence, it is in project root.
  • You can also have the $HOME/.cordova/.
  • Here, the folder will have configuration settings meant to be applied at user level ( and not at project level)
  • Will contain a file called config.json
  • You can add hooks folder here, too, but it is deprecated.

config.json

  • Project specific configuration for cordova command.
  • Sets project name and other settings

.git folder

  • A typical git folder to hold the versioning

config.xml

  • Holds the application configurations
  • Arguably the most important file in the structure
  • Holds the name of the application
  • Holds the description of the application
  • Holds the icons and splashscreen information
  • Author's name and email
  • Various application preferences, e.g. device orientation
  • Lists plugins

hooks folder

  • Contains special scripts that are executed on occurance of specific event
  • That way, a script is 'hooked' to an event.
  • Events like after_build, after_compile, before_run etc

platforms folder

  • Contains Platform specific project
  • A new subfolder for each platform is created

plugins folder

  • Stores the plugins

CREATING APPS WITH BUILD SERVICE

Follow the steps below.

phonegap create phonegap-buildservice-helloworld

cd phonegap-buildservice-helloworld

git init

git add .

git commit -m 'Init commit'

Create a repository on Github. Set the remote.

git push origin master

CREATING APPS WITH BUILD SERVICE

  • Link your github account to your build service account
  • Go to https://build.phonegap.com/apps
  • Click on '+ new app'
  • Provide the reference to git repo you created or upload a zip file of the project folder
  • It will fetch the code and provide you with further options
  • Click on 'Ready to Build'
  • You will see app getting build for various platforms
  • Download the app bundle and test it. You can also scan the barcode to download the app.

MAKE SOME CHANGES

  • Open index.html and edit the content
  • Create another commit with this change
  • Push the commit to the Github repo
  • Go to Build Service Portal
  • Select your app by clicking its title or icon.
  • Pull the latest code from your Github repo. You need to do it under the 'Settings' section of your app.
  • Rebuild. Download. Test.

Understanding Hooks

  • Dedicated scripts that are invoked on a particular event
  • These scripts can be added by application developer and/or plugin developer
  • Plenty of events, allowing finer control at every step of development workflow
  • Three different way exists to introduce hooks to your project
